# Prompt de Mejoras Arquitectónicas y de Código

Copia y pega este prompt al asistente o agente cuando desees iterar mejoras de código en el proyecto:

---

Eres un desarrollador experto en WordPress y PHP 8. Revisa e implementa las siguientes mejoras arquitectónicas en el plugin `Woo OTEC Moodle` asegurándote de no romper las integraciones actuales.

El objetivo principal en esta etapa es **estabilizar completamente el plugin, mejorar su arquitectura interna y asegurar funcionamiento correcto en producción antes de agregar sistemas comerciales (licencias, updates, etc.)**.

---

## 1. **[PRIORIDAD ALTA] Consistencia de Nombres y Estructura**

* El archivo principal es `woo-otec-moodle.php`
* El Text Domain debe ser `woo-otec-moodle`
* Todas las constantes, clases y prefijos deben estar alineados
* El README debe reflejar la estructura real (`includes/`, no `core/`)

**Tarea:**

* Unificar naming en todo el plugin
* Revisar includes, rutas, hooks y autoload
* Eliminar inconsistencias históricas

---

## 2. **[PRIORIDAD ALTA] Estabilidad General del Plugin**

* Validar que el plugin:

  * Se active sin errores
  * No genere warnings/notices
  * No rompa el admin de WordPress
  * No genere errores AJAX
  * No tenga conflictos con WooCommerce

**Tarea:**

* Revisar hooks (`add_action`, `add_filter`)
* Validar nonces en AJAX
* Sanitización y escaping completo (`sanitize_text_field`, `esc_html`, etc.)
* Manejo de errores consistente

---

## 3. **[PRIORIDAD ALTA] Refactorización de `class-admin.php`**

Actualmente es una clase monolítica.

**Tarea: dividir en:**

* `admin/class-settings.php`
* `admin/class-ajax-handlers.php`
* `admin/class-tools.php`

**Objetivo:**

* Aplicar SRP (Single Responsibility Principle)
* Mejorar mantenibilidad
* Reducir acoplamiento

---

## 4. **[PRIORIDAD MEDIA] Mejora de Arquitectura (Core del Plugin)**

* Reducir uso innecesario de Singletons
* Centralizar carga en `class-core.php`

**Tarea:**

* Implementar inicialización ordenada de servicios
* Inyección de dependencias cuando sea posible
* Evitar instancias globales innecesarias

---

## 5. **[PRIORIDAD MEDIA] Manejo de Errores de Moodle**

Actualmente los errores están dispersos.

**Tarea:**

* Crear clase `PCC_Moodle_Exception`
* Implementar manejo con `try/catch`
* Centralizar errores en `class-sync.php`

---

## 6. **[PRIORIDAD MEDIA] Limpieza y Optimización General**

* Eliminar código muerto
* Eliminar duplicaciones
* Revisar includes innecesarios
* Optimizar consultas y procesos

---

## 7. **[PRIORIDAD MEDIA] Validación de Estructura del Plugin**

Estructura esperada:

woo-otec-moodle/
├── admin/
├── includes/
├── public/
├── assets/
├── woo-otec-moodle.php

**Tarea:**

* Validar que todo esté correctamente organizado
* Evitar archivos fuera de contexto
* Mantener estructura limpia y profesional

---

## 8. **[PRIORIDAD BAJA - POSTERIOR] Sistema de Licencias y Auto-Update**

⚠️ **IMPORTANTE: ESTE PUNTO NO DEBE IMPLEMENTARSE TODAVÍA**

Se implementará SOLO cuando el plugin esté:

* Estable
* Funcional
* Sin errores
* Listo para producción

En ese momento se integrará:

* Sistema de licencias
* Validación por dominio
* Auto-update remoto
* API externa

* Refactorizar sobre el código existente

* Mantener el plugin compacto, claro y mantenible

---

## 10. **Regla Permanente para Pruebas**

Después de cada mejora relevante:

* Generar el ZIP instalable:

/plugins/woo-otec-moodle.zip

* Mantener backups automáticos
* Validar instalación real en WordPress

---

## 11. **Regla CRÍTICA de ZIP (Compatibilidad WordPress)**

Siempre asegurar:

str_replace('\', '/', $path);

Antes de agregar archivos al ZIP.

Motivo:

* WordPress falla al extraer rutas con `\`
* Solo acepta `/` como separador válido
